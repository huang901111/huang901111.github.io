Python HTTP代理

Aug 3, 2017 #Python

介绍几种通过urllib/requests模块使用HTTP代理方法

一、HTTP代理介绍

  • HTTP代理 最常见用途是用于网络共享、网络加速和网络限制突破;此外,HTTP代理也常用于Web应用调试，Android/IOS APP中所调用的Web API监控和分析。
  • HTTP代理 本质上是一个web应用，和其他普通web应用没有根本区别。

1.1 HTTP 代理工作原理

  1. HTTP代理收到请求后，根据header中host字段的主机名和Get/Post请求地址综合判断目标主机
  2. 建立新的HTTP请求并转发请求数据
  3. 将收到的响应数据转发给客户端

注意：在无法设置HTTP代理的情况下，最简单的方式就是将目标Host域名的IP指向代理服务器，可以采取修改hosts文件的方式实现

二、Python程序中设置HTTP Proxy

2.1 urllib/urllib2 代理设置

  • urllib2 是通过ProxyHandler来是指代理服务器
import urllib2

proxy_handler = urllib2.ProxyHandler({'http':'xxx.xxx.xxx.xxx:8111'}) # 设置HTTP代理的IP和端口
opener = urllib2.build_opener(proxy_handler)
r = opener.open('http://www.baidu.com')
print r.read()
  • 可以用install_opener将配置好的opener安装到全局环境中，这样所有的urllib2.urlopen都会自动使用代理
import urllib2

proxy_handler = urllib2.ProxyHandler({'http':'xxx.xxx.xxx.xxx:8111'}) # 设置HTTP代理的IP和端口
opener = urllib2.build_opener(proxy_handler)
urllib2.install_opener(opener)
r = urllib2.urlopen('http://www.baidu.com')
print r.read()

2.2 requests 代理设置

  • requests 是目前最优秀的HTTP库之一，平时构造http请求时使用最多的库
import requests

urlread = requests.get('http://www.baidu.com',proxies={'http':'xxx.xxx.xxx.xxx:8110'}).json
print urlread
  • 可以直接设置session的proxies属性，省去每次请求都带上proxies参数的麻烦
import requests

s = requests.session()
s.proxies = {'http':'xxx.xxx.xxx.xxx:80'}
print s.get('http://www.baidu.com').json()

2.3 HTTP_PROXY/HTTPS_PROXY 环境变量

  • urllib2 和 requests 库都能识别HTTP_PROXY和HTTPS_PROXY环境变量，一旦检测到这些环境变量就会自动设置使用代理。
  • 大部分软件都支持HTTP_PROXY环境变量识别
http_proxy="xxx.xxx.xxx.xxx:xxx"
