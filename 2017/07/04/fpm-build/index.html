FPM 命令

Jul 4, 2017 #Linux

自动化部署必备技能-定制化RPM包

一、FPM 简介

  • FPM功能简单说就是一种类型的包转换成另外一种类型

支持的源类型包

  1. dir将目录打包成所需要的类型，可以用于源码编译安装的软件包
  2. rpm对rpm进行转换
  3. gem 对rubygem包进行转换
  4. python 将python模块打包成相应的类型

支持的目标类型包

  1. rpm 转换为rpm包
  2. deb转换为deb包
  3. solaris 转换为solaris包
  4. puppet 转换为puppet包

二、FPM 部署

  • 环境：CentOS release 6.2 (Final)
  • 软件包：Nginx-1.8.1 （提前源码安装完毕）

2.1 依赖包与模块安装

# yum -y install gcc gcc-c++ make ruby rubygems ruby-devel rpm-build
'### 需要ruby模块支持'
# gem sources -a http://mirrors.aliyun.com/rubygems/
'### 添加阿里云rubygems仓库，由于国外经常抽风..支持国货'
# gem sources --remove http://rubygems.org/

三、利用FPM制作Nginx RPM包

3.1 创建Nginx打包代码虚拟目录

mkdir -p /data/nginx/{etc/rc.d/init.d,sysconfig}
# etc/rc.d/init.d 用于保存 Nginx 启动脚本
cp -a /etc/init.d/nginx /data/nginx/etc/rc.d/init.d/
mkdir -p /data/nginx/usr/local
# usr/local 用于保存 Nginx 配置
cp -a /usr/local/nginx /data/nginx/usr/local/
# 将Nginx包目录拷贝到虚拟目录中

3.2 编写RPM包脚本

cd /data/nginx
cat >before_install_nginx.sh<<EOF
#!/bin/bash
# before install nginx
if rpm -q httpd &>/dev/null;then
        rpm -e --nodeps httpd
fi
if ! grep "nginx" /etc/group &>/dev/null;then
        groupadd nginx
fi
if ! grep "nginx" /etc/passwd &>/dev/null;then
        useradd  -g nginx -s /sbin/nologin -M nginx
fi
exit 0
# EOF
# 以上脚本主要是 Nginx RPM 包安装前执行的操作

cat >after_install_nginx.sh<<EOF
#!/bin/bash
# after install nginx
chkconfig --add nginx
chkconfig nginx on
service nginx start
exit 0
# EOF
# 以上脚本主要是 Nginx RPM 包安装后执行的操作

cat >after_remove_install.sh<<EOF
#!/bin/bash
# after remove nginx
nginx_pid=(/bin/grep−oE[0−9]+<<<(/bin/grep−oE[0−9]+<<<(awk '{print 7}' <<<7}' <<<(netstat -tunlp | grep nginx )))
kill ${nginx_pid}
chkconfig nginx off
chkconfig --del nginx
rm -rf /usr/local/nginx
rm -rf /etc/init.d/nginx
rm -rf /etc/rc.d/init.d/nginx
userdel -rf nginx
exit 0
# EOF
# 以上脚本主要是 Nginx RPM 包卸载后执行的操作

3.3 执行 FPM 打包操作

fpm -s dir -t rpm \
# -s 指定需要转换的源包类型
# -t 指定目标类型，即需要制作什么的包
-n nginx -v 0.0.1 \
# -n 定义软件包的名字
# -v 定义软件包的版本号
# --epoch 0 --iteration 1.el6 \
# --epoch 为软件包设置 epoch 值
# --iteration 就是rpm包中的release号
# -C /data/nginx/ -p /data/backup/ \
# -C 指定源码打包的相对路径
# -p 输出安装包的目录
-d 'openssl' -d 'openssl-devel' \
# -d 指定依赖于哪些安装包
-d 'gcc' -d 'gcc-c++' -d 'pcre' -d 'pcre-devel' \
-d 'zlib' -d 'zlib-devel' \
--verbose --category 'Development/Languages' \
# --verbose 输出制作过程
# --category RPM 包安装后软件包的归属组信息
--description 'nginx 1.8.1 applications' \
# --description 定义软件包的描述
--url 'nginx.org' --license 'BSD' -m 'Seacme' \
# --url 定义软件的官网url信息
# --license 定义许可证
--no-rpm-sign \
--pre-install /data/nginx/before_install_nginx.sh \
# 定义安装 Nginx RPM 包之前需要执行的操作
--after-install /data/nginx/after_install_nginx.sh \
# 定义安装 Nginx RPM 包之后需要执行的操作
--after-remove /data/nginx/after_remove_install.sh \
# 定义卸载 Nginx RPM 包之后要执行的操作
--workdir /data/nginx/ etc usr
# 定义利用 FPM 打包的目录
