Shell条件判断

Jul 24, 2017 #Linux

日常条件判断倡导使用[[，支持字符串模式匹配与正则表达式

一、认识test、[、[[

# `test`和`[`属于Shell的内建命名，`[[`属于Shell的保留关键字
type 'test' '[' '[['
test is a shell builtin
[ is a shell builtin
[[ is a shell keyword

1.1 [] 单双括号 与 test

  • 基本要素

    • [] 实际上是bash中test命令的简写，即所有的 [ expr ] 等于 test expr
    • [] 两个符号左右都要有空格分隔
    • 内部操作符与操作变量之间需要有空格：如 [ “a” = “b” ]
    • 字符串比较，> < 需要转义成>，<
    • [] 中字符串或者${}变量尽量使用”“双引号，避免值未定义引用而出错
    • [] 中可以使用 -a、-o 进行逻辑运算

  • 注意：

    • 比较运算符只有==和!=，两者都是用于字符串比较的，不可用于整数比较，整数比较只能使用-eq、-gt这种形式
    • 字符串比较还是整数比较建议不要使用大于号小于号，当需要使用的时需要使用『\』对大于号小于号进行转义

1.2 [[]] 双方括号

  • 基本要素

    • [[]] 两个符号左右都要有空格分隔
    • 内部操作符与操作变量之间要有空格：如 [[ “a” = “b” ]]
    • [[]] 中字符串或者${}变量尽量如未使用”“双引号括住的话，会进行模式和元字符匹配
    • [[]] 内部可以使用 && 、||进行逻辑运算

  • 注意：

    • 使用[]和[[]]时候不要吝啬空格，每一项两边都要有空格，如 [[ 1 = 2 ]]

1.3 [] 与 [[]] 比较

  • [[]] 运算符只是[]运算符的扩充。[[]]能够支持<、>符号运算不需要转义符，[]中必须转义>、<
  • [[]] 是以字符串比较大小，支持逻辑运算符：&&、||，不再使用 -a 、-o
  • bash中没有对[[]]中-eq操作符两边的内容进行检查对比，因为bash会直接把非整数的字符串转化为0
[] [[]]
数字测试 -eq、-ne、-lt、-le、-gt、-ge 同[]
文件测试 -r、-l、-w、-x、-f、-d、-s、-nt、-ot 同[]
字符测试 ==、!=、-n、-z，不可使用>=、<= 同[]
逻辑测试 -a、-o、！ &&、11(两竖)、！
数学运算 不可以使用 +、-、*、/、%
<> 比较结果异常 根据相应ASCII
\>、\< 根据相应ASCII码比较 不可以使用

1.4 [[]] 比[] 具备优势

  • [[是bash程序语言的关键字，并不是一个命令，[[]]结构比[]结构更加通用，在[[]]之间所有的字符都不会发生文件名扩展或者单词分割，但是会发生参数扩展和命令替换
  • 支持字符串的模式匹配，使用=~操作符时甚至支持shell的正则表达式。字符串比较时可以把右边的作为一个模式，而不仅仅是一个字符串。[[]]中匹配字符串或通配符，不需要引号
  • 使用[[]]条件判断结构，而不是[]，能够防止脚本的许多逻辑错误
  • bash把双中括号的表达式看作一个单独的元素，并返回一个退出码

二、文件比较运算符

运算符 描述
-e filename 如果filename存在，则为真
-d filename 如果filename为目录，则为真
-f filename 如果filename为常规文件，则为真
-L filename 如果filename为符号链接，则为真
-r filename 如果filename为可读，则为真
-w filename 如果filename为可写，则为真
-x filename 如果filename为可执行，则为
filename1 -nt filename2 如果filename 1 比 filename2 新，则为真
filename1 -ot filename2 如果filename1 比 filename2 旧，则为真

三、字符串比较运算符

运算符 描述
-z string 如果string长度为零，则为真
-n string 如果string长度为非零，则为真
string1 = string2 如果string1与string2相同，则为真
sting1 != string2 如果string1与string2不同，则为真

四、算数比较运算符

运算符 描述
num1 -eq num2 等于
num1 -ne num2 不等于
num1 -lt num2 小于
num1 -le num2 小于或等于
num1 -gt num2 大于
num1 -ge num2 大于或等于
