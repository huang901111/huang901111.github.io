Curl Command

Jul 24, 2017 #Linux

一个利用URL规则在命令行下工作的文件传输工具

一、Curl 简介

  • Curl命令是一个利用URL规则在命令行下工作的文件传输工具，支持文件的上传和下载
  • Curl命令支持包括Http、Https、Ftp等众多协议，还支持Post、Cookies、认证，从指定位置处下载文件
  • 支持限速、文件大小、进度条、断点续传功能

二、Curl 日常使用

2.1 将网页保存为指定文件名的文件

# 利用『-o』选项将文件下载到指定命名『mygettext.html』文件中
curl -o /data/backup/tmp/mygettext.html http://www.gnu.org/software/gettext/manual/gettext.html

2.2 将网页直接保存到本地

# 利用『-O』选项将网页直接保存到当前路径下
curl -O http://www.gnu.org/software/gettext/manual/gettext.html
# 同样可以使用">"执行重定向，与『-o』选项功能一致
curl http://www.gnu.org/software/gettext/manual/gettext.html > /data/backup/tmp/mygettext.html

2.3 同时下载多个网页到本地

# Curl 命令同时跟上『-O』选项实现多个网页同时下载
curl -O URL1 -O URL2

2.4 断点续传

# 当文件在下载完成之前结束该进程
curl -O http://www.gnu.org/software/gettext/manual/gettext.html
##############             20.1%

# 通过添加『-C』选项继续对该文件进行下载，已经下载过的文件不会被重新下载
curl -C - -O http://www.gnu.org/software/gettext/manual/gettext.html
###############            21.1%

2.5 网络限速

# 利用『--limit-rate』选项对网络使用进行限制
curl --limit-rate 1000B -O http://www.gnu.org/software/gettext/manual/gettext.html

2.6 采用代理访问

# 指定代理主机和端口,指定122.224.210.148主机和8383端口访问
curl -x 122.224.103.148:8383 http://google.com.cn

2.7 打印表头信息

2.8 静默下载

# 利用『-s』或者『--silent』选项实现静默下载,不显示下载进度条
curl -s -O http://www.gnu.org/software/gettext/manual/gettext.html

2.9 格式化网页状态信息

# 使用『-w』选项格式化输出一些用户操作的数据信息
# 格式化输出符包括"\t","\r","\n","@","%"
# "@filename"表示输出filename文件中的数据，"@-"表示输出用户写入标准输入的数据
# "%"标识符后面跟上定义的关键字，则可以输出对应的curl数据
- http_code:上次操作返回的响应码
- time_total：上次操作耗时，单位为秒
- time_connect:TCP连接2远程主机的耗时，单位为秒
- time_pretransfer:文件真正开始传输之前交互耗时，单位为秒
- time_starttransfer:文件的第一个字节开始出传输之前的耗时，单位为秒
- size_download:下载的总字节数
- size_upload:上传的总字节数
- size_request:发送http请求的总字节数
- speed_download:curl成功下载的平均下载速度
- speed_upload:curl成功上传的平均下载速度
curl -s -w %{time_connect}"\n"%{time_starttransfer}"\n"%{total}"\n" -O http://www.gnu.org/software/gettext/manual/gettext.html
0.284
0.576
10.247

三、Curl 实现邮件发送

3.1 确认curl是否支持smtp

curl-config --protocols | /bin/grep smtp

3.2 源码安装高版本curl

  • curl下载链接
  cd /data/backup/tmp
  wget https://curl.haxx.se/download/curl-7.50.3.tar.gz
  tar -zxf curl-7.50.3.tar.gz
  cd curl-7.50.3
  ./configure --prefix=/usr/local/curl
  make && make install
  mv /usr/bin/curl /usr/bin/curl.bak
  mv /usr/bin/curl-config /usr/bin/curl-config.bak
  ln -s /usr/local/curl/bin/curl /usr/bin/curl
  ln -s /usr/local/curl/bin/curl-config /usr/bin/curl-config
  • 检查该版本curl是否支持smtp
  curl-config --protocols | /bin/grep SMTP
  SMTP
  SMTPS

3.3 使用Curl发送邮件测试

  • 编写Curl邮件发送脚本
  #!/bin/bash
  # write by Seacme Huang

  if [ ${#} -lt 3 ];then
      echo -e "邮件接口：
  sh ${0} 收件人 邮件主题 邮件正文"
      exit 100
  fi

  # 设置发送报警的邮箱
  Mail_From="xxxxxx@163.com"
  Mail_To=${1}
  Mail_Subject=${2}
  Mail_Content=${3}
  Mail_Content_File="/data/tmp/curl-mail-$(date +%Y%m%d).txt"
  Mail_Smtp="smtp.163.com"
  Mail_User="xxxxxx@163.com"
  Mail_Password="xxxxxxx"

  # 判断邮件正文是否文件，若是文件，则读取，反之直接输入
  if [ -f ${Mail_Content} ];then
      Content=`cat ${Mail_Content}`
  else
      Content=`echo ${Mail_Content}`
  fi 

  # 判断是否存在${Mail_Content_File}
  [ -f ${Mail_Content_File} ] || touch ${Mail_Content_File}

  # 创建${Mail_Content_File}
  echo "From:${Mail_From}
  To:${Mail_To}
  Subject:${Mail_Subject}

  ${Content}" >> ${Mail_Content_File}

  # 执行邮件发送操作
  Curl=`which curl`
  ${Curl} -s -url "${Mail_Smtp}" --mail-From "${Mail_From}" --mail-rcpt "${Mail_To}" --upload-file ${Mail_Content_File}  --user "${Mail_User}:${Mail_Password}"
  if [ $? -eq 0 ];then
      echo -e "$(date +%c) Info 发送邮件到${Mail_To},\e[1;32m OK \e[0m"
  else
      echo -e "$(date +%c) Error 发送邮件到${Mail_To},\e[1;31m 异常 \e[0m"
  fi

  # 发送邮件后删除正文内容文档
  [ -f ${Mail_Content_File} ] && /bin/rm -f ${Mail_Content_File}
  exit 0
  • 举例操作
  sh curl-mail.sh "xxxxx@xxx.com" "Curl邮件测试" "/data/backup/tmp/mail_content.txt"
  Sat 17 Sep 2016 02:47:13 PM CST Info 发送邮件到xxxxx@xxx.com, OK
