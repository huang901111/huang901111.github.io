Python ConfigParser模块

Jul 19, 2017 #Python

Configparser用于读取配置文件

一、Configparser简介

  • Configparser用来读取配置文件的包。
  • 配置文件格式：中括号”[]“内包含的为section。

1.1 ini文件结构

注意事项：

  1. 键值对 可用”=“或者”:“进行分隔
  2. section 的名字是区分大小写的，而 key 的名字是不区分大小写的
  3. 键值对 中头部和尾部的空白符会被去掉
  4. 值可以为多行
  5. 配置文件可以包含注释，注释以”#“或者”;“为前缀

二、基本使用

2.1 ini文档模型

[OA]
IP = 192.168.8.58
Backup_Time = 9
Backup_Folder = /data/www/oa
Backup_Database = oa
Backup_Table = t.login

[test]
IP = 192.168.8.73
Backup_Time = 9
Backup_Folder = /data/www/test
Backup_Database = test
Backup_Table = t.login

2.2 基本操作

  • 读取配置文件

    1. read(filename)：直接读取配置文件内容
    2. readfp(fp)：读取配置文件,fp为文件操作类，可以用open(filename)来填充
    3. sections()：返回所有的section列表
    4. options(section)：返回section下的所有option列表
    5. items(section)：得到section下面的键对值
    6. get(section,option)：得到section中的option值，string类型返回
    7. getint(section,option)：返回类型int
    8. getboolean(section,option)：返回类型为bool
    9. getfloat(section,option)：返回类型为float

  • 修改配置文件

    1. add_section(section)：添加一个新的section
    2. set(section,option,value)：对section中的option进行修改
    3. write(fp)：将内容写入到配置文件，fp为文件操作类，可以用open(filename)来填充
    4. remove_section(section)：删除section
    5. remove_option(section,option)：删除option

  • 判断配置文件

    1. has_section(section)：判断ini文件中是否存在section
    2. has_option(section,option)：判断section中是否存在option

2.3 初始工作

  • 使用Configparser首选需要初始化实例，并读取配置文件
cf = ConfigParser.ConfigParser()
cf.read('配置文件')

2.4 常见用法

  • 获取所有的section列表
secs = cf.sections()
print secs
['OA', 'test']
  • 返回section下所有的option列表
opts = cf.options('test')
print opts
['ip', 'backup_time', 'backup_folder', 'backup_database', 'backup_table']
  • 返回section下面的键对值
its = cf.items('OA')
print its
[('ip', '192.168.8.58'), ('backup_time', '9'), ('backup_folder', '/data/www/oa'), ('backup_database', 'oa'), ('backup_table', 't.login')]
  • 返回section中的options值，string类型返回
ip_info = cf.get('OA','IP')
print ip_info
192.168.8.58
  • 对section中的options值进行修改
new_ip = cf.set('test','IP','192.168.8.72')
cf.write(open('xxxx.txt','w'))
print cf.get('test','IP')
192.168.8.7
  • 添加一个新的section
cf.add_section('test2')
cf.set('test2','phone','137112121212')
cf.write(open('xxxx.txt','w'))
print cf.items('test2')
[('phone', '13121212121')]
  • 删除一个section
cf.remove_section('test2')
cf.remove_option('phone')
cf.write(open('xxxx.txt','w'))
print cf.sections()
['OA', 'test']

三、ConfigParser高级功能

  • 三种读取配置文件的工具：ConfigParser、RawConfigParser、SafeConfigParser的异同点

3.1 配置文件例子

[db]
url = http://%(host)s:%(port)s/db
host = 1.2.3.4
port = 342

3.2 ConfigParser读取

cf = ConfigParser.ConfigParser()
cf.read('xxxx.conf')

print cf.get('db','url')
http://1.2.3.4:342/db

3.3 RawConfigParser读取

rcf = ConfigParser.RawConfigParser()
rcf.read('xxxx.conf')

print rcf.get('db','url')
http://%(host)s:%(port)s/db

3.4 SafeConfigParser读取

scf = ConfigParser.SafeConfigParser()
scf.read('xxxx.conf')

print scf.get('db','url')
http://1.2.3.4:342/db

总结：

  • RawConfigParser没有办法处理环境变量
  • ConfigParser与SafeConfigParse可以传递变量进行赋值
