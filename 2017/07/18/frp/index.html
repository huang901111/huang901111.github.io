FRP 工具

Jul 18, 2017 #Network

一款内网穿透高兴能反向代理

一、内网穿透原理

简单地说，内网穿透依赖于NAT原理，根据NAT设备不同大数可以分为以下4大类（前3种NAT类型可统称为cone类型）

  1. 全克隆（Full Cone）：NAT把所有来自相同内部IP地址和端口的请求映射到相同的外部IP地址和端口上，任何一个外部主机均可通过该映射反向发送IP包到该内部主机
  2. 限制性克隆（Restricted Code）：NAT把所有来自相同内容IP地址和端口的请求映射到相同的外部IP地址和端口；但是，只有当内部主动先给IP地址为X的外部主机发送IP包时，该外部主机才能向该内部主机发送IP包
  3. 端口限制性克隆（Port Restricted Code）：端口限制性克隆与限制性克隆类似，只是多了端口号的限制，即只有内部主机先向IP地址为X，端口为P的外部主机发送1个IP包，该外部主机才能够把源端口号为P的IP包发送给该内部主机
  4. 对称式NAT(Symmetric NAT)：这种类型的NAT与上述3种类型的不同，在于当同一个内部主机使用相同的端口与不同地址的外部主机进行通信时，NAT对该内部主机的映射会有所不同；对称式NAT不保证所有会话中的私有地址和公开IP之间绑定的一致性；相反，它为每个新的会话分配一个新的端口号；导致此种NAT根本无法穿透

二、认识frp

  • frp是一个可用于内网穿透的高性能的反向代理应用，支持tcp、udp、http、https协议

2.1 frp作用

  1. 利用处于内网或防火墙后的机器，对外网环境提供http或者https服务
  2. 对于http、https服务基于域名的虚拟主机，支持自定义域名绑定，使用多个域名可以共用一个80端口
  3. 利用处于内网或防火墙后的机器，对外网环境提供tcp和udp服务

三、frp部署

3.1 服务器端部署

  • 下载源码包
wget https://github.com/fatedier/frp/releases/download/v0.11.0/frp_0.11.0_linux_amd64.tar.gz
  • 部署服务端
cd /data/backup/tmp
# 解压源码包
tar -zxf frp_0.11.0_linux_amd64.tar.gz -C /usr/loca/frp
# 服务端配置
cd /usr/loca/frp
vim frps.ini
[common]
## frp 监听地址
bind_addr = 0.0.0.0
bind_port = 7000
## 代理http服务
vhost_http_port = 80
## frp 控制面板
dashboard_port = 7500
dashboard_user = xxxxxx
dashboard_pwd = xxxxxxxxxxx
## 默认日志输出位置(这里输出到标准输出)
log_file = /var/log/frp_server.log
## 日志级别，支持: debug, info, warn, error
log_level = info
log_max_days = 3
## 开启tcp多路复用
tcp_mux = true
## 后端连接池最大连接数
max_pool_count = 100
## 自定义二级域名
subdomain_host = xxxx.cn
## own test machine
[ssh-214]
type = tcp
auth_token = xxxxxxxxxxxx
bind_addr = 0.0.0.0
listen_port = 6168
## own test service
[web-owncloud]
type = http
auth_token = xxxxxxxxxxxx
  • 利用supervisor管理frp
## supervisor配置
cd /etc/supervisor/conf.d
vim frp_server.conf
[program:frp_server]
user=root
directory=/usr/local/frp
command=/usr/local/frp/frps -c "/usr/local/frp/frps.ini"
process_name=%(program_name)s
autostart=true
redirect_stderr=true
stdout_logfile=/var/log/frp_server.log
## 启动frp server
/etc/init.d/supervisord restart

3.2 客户端配置

  • 下载源码包
wget https://github.com/fatedier/frp/releases/download/v0.11.0/frp_0.11.0_linux_amd64.tar.gz
  • 部署客户端
cd /data/backup/tmp
## 解压源码包
tar -zxf frp_0.11.0_linux_amd64.tar.gz -C /usr/local/frp
## 客户端配置
cd /usr/local/frp
vim frpc.ini
[common]
server_addr = xxx.xxx.xxx.xxx
server_port = xxxx
log_file = /var/log/frp_client.log
log_level = info
log_max_days = 3
[ssh-123]
type = tcp
local_ip = 127.0.0.1
local_port = xxxx
remote_port = 6169
auth_token = W7gbxsB9HtYm4zwVwb
use_encryption = true
use_compression = true
[web-owncloud]
type = http
local_port = 80
auth_token = te9cuL5QnGxh4hnmVT
subdomain = owncloud
  • 启动客户端
cd /usr/local/frp
./frpc -c frpc.ini &
