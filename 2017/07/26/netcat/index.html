Netcat Command

Jul 26, 2017 #Linux

网络安全界『瑞士军刀』，透过使用TCP或者UDP协议去读写数据

一、NetCat 简介

  • NetCat 适用于任意TCP和UDP的连接与监听
  • NetCat 被誉为网络安全界的“瑞士军刀”，一个简单而有用的工具，透过使用TCP或UDP协议的网络连接去读写数据。
  • NetCat 被设计成一个稳定的后门工具，能够直接由其它程序和脚本轻松驱动。同时，它也是一个功能强大的网络调试和探测工具，能够建立你需要的几乎所有类型的网络连接。

二、NetCat 基本用法与日常用途

2.1 NetCat 基本用法

  • 通过nc想要连接到某处

    nc [-options] hostname port[s] [ports] …

  • 绑定端口等待连接

    nc -l -p port [-options] [hostname] [port]

  • 建立网络隧道

    nc -L hostnameort -p port [options]

  • 日常使用当中可使用 nc -h 获取相关帮助

2.2 NetCat 日常用途

  1. 简单的TCP代理
  2. 基于HTTP客户端与服务端之间 shell 脚本的执行
  3. 网络测试
  4. 作为SOCKS或HTTP代理服务提供给ssh操作

三、NetCat 日常使用参数

参数 作用 参数 作用
-D 启动debug套接选项 -h 打开帮助文档
-i secs 延迟端口扫描时间 -l 监听模式，用于入站连接
-n 指定数字的IP地址，不能用hostname -u UDP模式
-t TCP模式 -v 详细输出
-s 设置本地主机送出数据包的IP地址 -p 设置本地主机使用的通信端口
-w secs 设置timeout超时时间 -z 将输入输出关掉
-t TCP模式 -q 控制接收到 EOF 后隔多长时间才退出，该参数的单位是秒
  • 端口扫描可指定一个或者利用lo-hi的方式指定扫描范围

四、NetCat 具体实际应用操作

4.1 利用 NetCat 实现端口的扫描

### 基本格式：nc -z -v {host-name-here} {port-range-here}
nc -z -v -w 2 -n 192.168.8.xx 616xx 
Connection to 192.168.8.xx 616xx port [tcp/*] succeeded!
nc -z -v -w 2 -n 192.168.8.xx 61010-61618
### 实现某个范围段内端口扫描
nc -u -v -z -w2 192.168.1.xxx 1-30
### 指定-u实现UDP扫描

4.2 利用 NetCat 实现远程文件拷贝

  • server1：192.168.8.72 (Client端)
  • server2：192.168.8.73 （Server端）
### Client 端
nc -l 1536 >  /data/tmp/client.txt
### Server 端
echo "HelloClient" > /data/tmp/server.txt
nc 192.168.8.72 1567 < /data/tmp/server.txt
### Server端执行后，Client端client.txt会输入HelloClient

4.3 利用 NetCat 实现目录传输

  • server1：192.168.8.72 (Client端)
  • server2：192.168.8.73 （Server端）
### Server 端
tar -cvf - /data/tmp/tgzt_4399_999 | bzip2 -z | nc -l 1567
### Client 端
nc -n 192.168.8.73 1567 | bzip2 -d | tar -xvf -
### 利用tar打包文件夹发送，bzip2加压节省传输压缩包

4.4 利用 NetCat 克隆设备

  • server1：192.168.8.72 (Client端)
  • server2：192.168.8.73 （Server端）
### Server 端
dd if=/dev/sda | nc -l 1567
### Client 端
nc -n 192.168.8.73 1567 | dd of=/dev/sda
### dd是一个从磁盘读取原始数据的工具
### 通过netcat服务器重定向它的输出流到其他服务器并写入磁盘

4.5 利用 NetCat 创建远程shell

  • server1：192.168.8.72 (Client端)
  • server2：192.168.8.73 （Server端）
### Server 端
mkfifo /tmp/tmp_fifo
cat /tmp/tmp_fifo | /bin/bash -i 2>&1 | nc -l 1567 > /tmp/tmp_fifo
### Client 端
nc -n 192.168.8.73 1567
### 以上操作类似执行ssh或者telnet远程机器，执行命令

4.6 利用 NetCat 指定端口绕过防火墙设置

  • server1：192.168.8.72 (Client端)
  • server2：192.168.8.73 （Server端）
### Server 端
nc -l 1567
### Client 端
nc 192.168.8.73 1567 -p 25
### 假如客户端只开放25端口

4.7 利用 NetCat 实现简单聊天工具

  • server1：192.168.8.72 (Client端)
  • server2：192.168.8.73 （Server端）
### Server 端
nc -l 1567
### Client 端
nc 192.168.8.73 1567

4.8 利用 NetCat 连接远程主机

nc -nvv 192.168.8.72 61618

4.9 利用 NetCat 监听本机端口

nc -l -p 80 
### 监听本机80端口
