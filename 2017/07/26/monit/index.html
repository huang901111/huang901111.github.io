Monit监控

Jul 26, 2017 #Monitoring

一款跨平台轻量级监控，可全面、进程状态、文件系统监控与报警

一、Monit介绍

  • Monit是一个跨平台的用来监控unix/linux系统（比如Linux、BSD、OSX、Solaris）的工具。
  • Monit特别易于安装，而且非常轻量级，并且不依赖任何第三方程序、插件或者库。
  • Monit可以胜任全面监控、进程状态监控、文件系统变动监控，邮件通知和对核心服务的自定义动作等场景。

二、Monit 部署

2.1 下载Monit源码包

wget https://mmonit.com/monit/dist/monit-5.20.0.tar.gz

2.2 Monit 安装

cd /data/backup/tmp/
tar -zxf monit-5.20.0.tar.gz 
cd monit-5.20.0
./configure --prefix=/usr/local/monit
make && make install

2.3 Monit 配置

cp -a /data/backup/tmp/monit-5.20.0/monitrc /etc/
vim /etc/monitrc
set daemon  30  
# 设置monit作为守护进程运行，并且每30秒监控一次
    with start delay 120  
    # 加载配置2分钟后开始运行
set logfile /data/logs/monit.log  
# 设置日志文件的位置
set pidfile /var/run/monit.pid  
# 设置服务pid文件保存位置
set idfile /var/.monit.id 
# 设置存储monit实例的唯一ID
set eventqueue  
# 如果没设置email，会自动alert
    basedir /var/monit  
    # set the base directory where events will be stored
set mailserver smtp.163.com

三、Monit 监控设置

3.1 设置监控ssh服务

vim /etc/monit.d/check_sshd.cfg
check process sshd with pidfile /var/run/sshd.pid

   start program  "/etc/init.d/sshd start"

   stop program  "/etc/init.d/sshd stop"

   if failed port 61618 protocol SSH then restart

   if 5 restarts within 5 cycles then timeout

3.2 设置监控network

check network eth0 interface eth0
	start program = "/sbin/ifup eth0"
	stop program = "/sbin/ifdown eth0"
	if failed link then restart
	if saturation > 85% then alert
	if upload > 500 kB/s then alert
	if total downloaded > 1 GB in last 2 hours then alert
	if total downloaded > 10 GB in last day then alert

3.3 设置监控messages

check file message with path /var/log/messages
	if content = "rm" then alert
	if content = "authorized_keys" then alert
	if content = "iptables" then alert

3.4 设置监控文件系统

check filesystem root with path /
	if space usage > 80% then alert
	if inode usage > 75% then alert

check filesystem data with path /data
	if space usage > 80% then alert
	if inode usage > 75% then alert

check filesystem boot with path /boot
	if space usage > 80% then alert
	if inode usage > 75% then alert

3.5 设置监控系统

check system $HOST
	if loadavg(1min) > 2 then alert
	if loadavg(5min) > 3 then alert
	if loadavg(15min) > 1 then alert
	if memory usage > 80% for 4 cycles then alert
	if swap usage > 20% for 3 cycles then alert
	if cpu usage (user) > 80% for 2 cycles then alert
	if cpu usage (system) > 20% for 2 cycles then alert
	if cpu usage (wait) > 80% for 2 cycles then alert
	if cpu usage > 200% for 4 cycles then alert

3.6 设置监控mysql服务

check process mysql with pidfile /tmp/mysql.pid
	start program = "/etc/init.d/mysqld start"
	stop program = "/etc/init.d/mysqld stop"
	if failed port 3306 protocol mysql then restart
	depends on mysql_bin
	depends on mysql_rc

check file mysql_bin with path /usr/local/bin/mysqld
	if failed checksum then unmonitor
	if failed permission 755 then unmonitor
	if failed uid root then unmonitor
	if failed gid root then unmonitor

check file mysql_rc with path /etc/init.d/mysqld
	if failed checksum then unmonitor
	if failed permission 755 then unmonitor
	if failed uid root then unmonitor
	if failed gid root then unmonitor

四、Monit 监控细节

4.1 一般的操作

# Monit监控运行
monit -c /etc/monitrc
# Monit监控配置语法检查
monit -t

4.2 Monit命令选项

options options options
-c file 加载配置文件 -d n n秒后后台运行 -g name 指定服务组进行监控
-l logfine 指定日志文件 -p pidfile 指定pid文件 -s statefile 指定状态文件
-I 不在后台运行 –id 打印unique ID –resetid 重新设置unique ID
-t 检查配置语法 start all 启动所有服务 stop all 停止所有
restart all 重新所有服务 monitor all 监听所有服务 reload 重新加载配置
status 打印所有服务的状态 summary 所有服务状态整理 quit 杀掉监听进程
validate 检查所有需要启动的服务，若未启动则启动

4.3 Monit守护进程管理

# 添加为init进程，并启动
vim /etc/inittab
# Run monit in standard run-levels  
mo:2345:respawn:/usr/local/monit/bin/monit -Ic /etc/monitrc 

# 启动并验证
telinit q
ps -ef | grep "monit" | grep -v "grep"  
root 5655 1 0 07:34 ? 00:00:00 /usr/local/monit/bin/monit -Ic /etc/monitrc

# 结束进程
kill -9 5655
ps -ef | grep "monit" | grep -v "grep"  
root 13905 1 0 10:16 ? 00:00:00 /usr/local/monit/bin/monit -Ic /etc/monitrc

4.4 定义服务

  • 定义服务，就是定义监控对象

    • monit可以定义9中类型，包括process/file/fifo/filesystem/directory/system/program/network/host
    • process：建龙指定进程是否存活
    • file/fifo：/filesystem/directory：监控指定的文件或者目录是否存在
    • system：监控系统资源，包括CPU、内存和负载
    • program：监控指定程序的退出或者运行时间的长短
    • network：监控系统指定interface，包括up/down，带宽，误码率，流量，ping
    • host：监控其他主机，可以定义端口、tcp/udp、protocol（http/mysql/websocket）

  • 监控主机的web端口，因此用host类型

check host <service> address <host ip address>

4.5 服务测试

服务测试提供了测试监控对象状态的一种方法，根据状态不同，可以指定相应的动作

  • 通用语法
if <test> then <action1> else if success then <action2>

  • test几种类型

    • 状态失效，如网络不可达，进程丢失
    • 参数变化：如文件权限，文件md5，文件大小，日期等
    • 参数达到上限：如CPU>80%,磁盘空间大于85%

  • action 5种类型

    • alert：发邮件告警
    • restart/start/stop：重启/启动/停止 service。在监控进程时特别有用，先定义start、stop、restart的命令，当进程丢失时，启动相应service，恢复进程
    • exec：执行用户定义的命令或者脚本

4.6 典型例子

check process apache with pidfile /var/run/httpd.pid
       start program = "/etc/init.d/httpd start"  
       # start 命令
       stop program  = "/etc/init.d/httpd stop" 
       # stop 命令
       if cpu > 40% for 2 cycles then alert 
       # 如果2次轮询，进程cpu都大于40%，则告警
       if total cpu > 60% for 2 cycles then alert 
       # 如果2此轮询，系统cpu都大于60%，则告警
       if total cpu > 80% for 5 cycles then restart 
       # 如果5次轮询，系统CPU都大于80%，则重启
       if mem > 100 MB for 5 cycles then stop 
       # 如果5次轮询，内存mem都大于100MB，则stop
       if loadavg(5min) greater than 10.0 for 8 cycles then stop  
       # 如果8次轮询，5分钟平均负载都大于10，则stop

五、Monit 官方教程

Monit 官方wiki（https://mmonit.com/monit/documentation/monit.html#SYNOPSIS）

5.1 补充知识点

  • 服务轮询检查
# 每个轮询检查2次
check process nginx with pidfile /var/run/nginx.pid
       every 2 cycles
       
# 在工作时间早上8点至下午7点检查
check program checkOracleDatabase
        with path /var/monit/programs/checkoracle.pl
       every "* 8-19 * * 1-5"
       
# 在某些时间段不检查
check process mysqld with pidfile /var/run/mysqld.pid
       not every "* 0-3 * * 0"
  • 服务重启限制
# 若服务在少于3次轮询中重启了2遍则不纳入监控
if 2 restarts within 3 cycles then unmonitor
  • 容错机制
if failed port 80 for 3 times within 5 cycles then alert
check filesystem rootfs with path /dev/hda1
  if space usage > 80% for 5 times within 15 cycles then alert
  • 存在判断机制
check file with path /cifs/mydata
  if does not exist then exec "/usr/bin/mount_cifs.sh"
  • 文件大小机制
check file mydb with path /data/mydatabase.db
    if size > 1 GB then alert

                  maxConnections="25"

6.3 数据库处理

create database mmonit
grant all privileges on mmonit.* to mmonit@127.0.0.1 identified by  mmonit;

flush privileges;
# 导入表结构
mysql -ummonit -pmmonit  mmonit < /usr/local/mmonit/db/mmonit-schema.mysql
