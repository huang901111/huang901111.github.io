NetHogs Trickle Command

Jul 15, 2017 #linux

Linux 查看进程流量与限速工具

NetHogs命令

  • NetHogs是一个小型的net top工具。
  • 不像大多数工具那样拖慢每个协议或者是每个子网的速度，而是按照了进程进行带宽分组。
  • NetHogs不需要依赖载入某个特殊的内核模块。
  • 如果发生了网络阻塞，可以启动NetHogs看到哪个PID造成这种状况。
  • 简单来说，可以通过NetHogs查看Linux下进程的流量

1.1 安装依赖包

  yum -y install libpcap libpcap-devel

1.2 安装NetHogs工具

  git clone https://github.com/raboof/nethogs
  cd nethogs
  make && make install && hash -r

1.3 NetHogs实例

  • 设置每5秒刷新一次
  nethogs -d 5
  • 监控eth0网络带宽
  nethogs eth0
  • 同时监控eth0和eth1网络带宽
  nethogs eth0 eth1
  • 指定网络带宽输出格式
  nethogs -v 1 -d 5
  # -v 指定网络带宽输出格式，默认值为0
  # 0 = KB/s, 1 = total KB, 2 = total B, 3 = total MB
  • 指定以发送流量排序输出
  nethogs -s -d 5
  • 检查nethogs工具版本
  nethogs -V

Trickle命令

  • Trickle是一款带宽控制供给，用来限制如Firefox，FTP,SSH以及其他使用网络带宽的程序的带宽。

  • Trickle可以让你基于单个应用程序来控制网络流量速率，而不是仅仅针对单个用户。

  • 补充：

    • Trickle通过控制socket数据读写量来控制和限制应用的上传/下载速度。

    • 注意:Trickle不能用于限制使用UDP协议的应用的带宽，它只可用于TCP连接，并且不能对所有的TCP有效，它只对使用Glibc动态库的应用有效。

1.1 安装Trickle工具

  • epel源安装
  yum -y install trickle

1.2 检查命令是否能以trickle限制

  ldd $(which rsync) | grep libc.so
  # 出现以下内容表示之处
  libc.so.6 => /lib64/libc.so.6 (0x00000031ec000000)

1.3 基本用法

  • 选项

    • -V：查看trickle工具的版本
    • -d：限制下载速度
    • -u：限制上传速度
    • -s：在独立模式下独立运行
    • -n：指定trickled socket名称保存文件。默认为/tmp/.trickled.sock

  • 用法一：限制wget下载速度

  trickle -s -d 100 wget "http://www.centoscn.com/plus/download.php?open=2&id=7341&uhash=899beee900c0a763e4402ab8"
  ......
  0% [                                       ] 1,128,268   99.8K/s  eta 1h 41m
  • 用法二：限制scp推送速度
  /usr/bin/trickle -s -d 50 scp -P61618 root@192.168.8.73:/data/backup/tmp/mc-curl-7.46.0-1.el6.x86_64.rpm ./
  • 用法三：限制rsync推送速度
  /usr/bin/trickle -s -d 50 rsync -avPe "ssh -p61618" root@192.168.8.73:/data/backup/tmp/mc-curl-7.46.0-1.el6.x86_64.rpm ./
