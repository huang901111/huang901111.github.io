Git Command

Jul 10, 2017 #linux

Git 一款分布式版本控制工具

配置身份信息

git config --global user.email "you@example.com"
git config --global user.name "your name"

将Git服务器代码克隆到本地仓库

git clone https://github.com/johcon/one.git

添加到本地缓存

git add .

查询缓存状态

git status

提交到缓存

git commit -m 'add xxxx'

推送到git分支

# 推送到master分支
git push origin master
# 推送到testing分支
git push origin testing

添加git标签一个1.0版本

git tag -a v1.0

删除本地文件与git文件

git rm -rf ok.txt

删除最近1次提交历史记录

git reset --hard HEAD^1
# HEAD 指向的版本就是当前版本，git允许我们在版本的历史之间穿梭，使用命令git reset --hard commit_id
# 穿梭前，用git log可以查看提交历史，以便确定要回退到哪个版本
# 要重返未来，用git reflog 查看命令历史，以便确定要回到未来的哪个版本

将缓存区中的修改移除

git reset --mixed HEAD
# 将缓存区中的修改移除，但改动还保留在工作目录中
git reset --hard HEAD
# 完全舍弃没有提交的改动，包含工作目录、缓存区

丢弃工作区的修改

git checkout -- filename.txt
# 当修改了工作区file文件内容，可执行git checkout -- filename.txt将修改操作回滚
# 当删除了工作区file文件，可执行git checkout -- filename.txt 将文件恢复

查看分支历史

git log --graph --pretty=oneline --abbrev-commit
# 合并分支时，加上--no-ff参数就可以使用普通模式合并，合并后的历史有分支，能看出来曾经做过合并
# fast forward合并就看不出来曾经做过合并

记录当前分支下每一次命令

git reflog

更新本地仓库最新改动

git pull origin master

分支操作

# 创建分支test并切换
git checkout -b test
# 列出分支
git branch
# 列出远程分支
git branch -r
# 将test分支推送到远端仓库
git push origin test
# 切换回主分支
git checkout master
# 比较分支的差异
git diff --stat master testing
# 将新建的分支删除
git branch -d testing

合并分支

# 将master分支合并到你的当前分支
git merge master

比较同一个文件在不同分支下区别

git diff master testing filename.txt

比较缓存区与工作目录文件差异

git diff --staged filename.txt

远程仓库操作

追加commit提交

# 情景：先修改执行commit操作
# 第一次commit操作
git commit -m 'add test操作'
# 添加文件
git add addfile.txt
# 追加到上一次commit操作中
git commit --amend
# 修改commit message描述内容
git commit --amend

删除远程分支和tag

# 删除远程分支
git push origin --delete {branch name}
# 删除tag
git push origin --delete tag {tag name}

远程主机更新，拉回本地

# 拉回远程仓库所有分支
git fetch
# 拉回远程仓库的master分支
git fetch origin master
# 查看远程分支
git branch -r 
# 查看所有分支
git branch -a 
# 拉回本地后与本地分支执行合并操作
git merge origin/master
# git pull 操作
git pull origin master:master 
## 效果等于
git fetch origin master
git merge origin/master

重命名远程分支

# 删除远程分支
git branch --delete origin {old_branch_name}
# 重命名本地分支
git branch -m {old_branch_name} {new_branch_name}
# 推送本地分支到远程
git push origin {new_branch_name}
