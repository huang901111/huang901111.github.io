Babun介绍

Jul 17, 2017 #Windows

一款即开即用的Windows Shell

一、Babun介绍

1.1 什么是Babun？

  • babun是windows上的一个第三方shell，在这个shell上面你可以使用几乎所有linux，unix上面的命令，他几乎可以取代windows的shell。

1.2 Babun 特点

  • 使用babun无需管理员权限
  • 先进的安装包管理器
  • 预先配置了Cygwin和很多插件
  • 拥有256色的兼容控制台
  • HTTP(S)的代理支持
  • 面向插件的体系结构
  • 可以使用它配置git
  • 集成了oh-my-zsh
  • 自动升级
  • 支持shell变成，内置VIM编辑器等

二、Babun安装部署

  • babun 下载

2.1 Babun 安装

## 默认执行 install.bat 安装到 『%userprofile%\.babun』目录中
## 需求：在C盘根目录中安装Babun
  • 拷贝『Babun压缩包』到『C盘根目录中』，解压『Babun压缩包』到当前目录下
  • 利用『cmd』模式安装『Babun』
# Win键 + R键 输入cmd确认后进入
# cd C:\babun-1.2.0\
# install.bat \t "C:\babun"
# 指定安装到C盘根目录下的babun文件夹
  • 安装完毕后，执行检查命令
babun check # 用于判断环境是否正确
babun update # 用于判断是否有更新包
  1. 使用 windows 用户本地认证
# 使用windows用户组
mkgroup -l > /etc/group 
# 使用windows用户的密码
mkpasswd -l > /etc/passwd

四、SSH服务安装配置

4.1 SSH服务配置

  • SSH配置文件：/etc/sshd_config
  • 配置操作如下
cd /etc
# 修改 sshd_config 文件权限
chmod 666 sshd_config
vim sshd_config
# 修改ssh服务端口
Port 61618
# 禁止root登陆
PermitRootLogin no
# CYGWIN=ntsec时的安全配置
StrictModes yes
# 禁止 rhosts 认证
RhostsRSAAuthentication no
# 禁止 rhosts 认证
IgnoreRhosts yes         
# 禁止密码认证
PasswordAuthentication no
# 禁止密码认证
ChallengeResponseAuthentication no
# 禁止空密码用户登录
PermitEmptyPasswords no
# 再次修改 sshd_config 文件权限
chmod 644 sshd_config

4.2 启动SSH服务

  • 服务启动命令
cygrunsrv -S sshd
  • 问题1
  cygrunsrv: Error starting a service: QueryServiceStatus:  Win32 error 1062:
  # 以上错误导致sshd服务启动失败，观察/var/log/sshd.log,发现错误
  Permissions 0650 for '/etc/ssh_host_ecdsa_key' are too open.
  It is recommended that your private key files are NOT accessible by others.
  This private key will be ignored.
  key_load_private: bad permissions
  Could not load host key: /etc/ssh_host_ecdsa_key
  # 发现是由于/etc/ssh_host_ecdsa_key文件权限导致
  # 解决方案
  chmod 600 /etc/ssh_host_ecdsa_key

4.3 配置key登录文件

  • Key保存目录：~/.ssh/authorized_keys
mkdir ~/.ssh/authorized_keys
# 创建key认证文件
cat > ~/.ssh/authorized_keys << EOF
......(ssh_key)
EOF
# 修改~/.ssh/authorized_keys
chmod 644 ~/.ssh/authorized_keys

五、其他软件安装与环境调整

5.1 软件安装

  • 本次部署涉及安装软件zip、svn、screen、subversion、until-linux、bind-utils、php
  • Babun 软件安装主要利用：pact
  • 举例说明：『zip软件安装』
pact install zip # Zip 软件安装
pact show | /bin/grep zip # 确认zip是否成功安装
  • 举例说明：『screen软件安装』
pact install screen # Screen 软件安装
pact show | /bin/grep screen # 确认screen是否成功安装
screen -S test # 启动测试
Directory /tmp/uscreens/S-cyg_server must have mode 700.
# 以上错误提示，必须将/tmp/uscreens/S-cyg_server权限修改为700
  • 举例说明：『subversion软件安装』
pact install subversion # Svn 客户端软件安装
  • 举例说明：『util-linux软件安装』『bind-utils软件安装』
pact install util-linux # 安装Linux基础工具
pact install bind-utils # 安装dig工具
  • 举例说明：『PHP软件安装』
pact install php php-curl php-json

5.2 修复终端中文异常

  • 设置终端字符集
终端 ——> 『Options』——> 『Text』——> 『Character Set』——> 『GBK』
  • 设置主题提示符更换
vi ~/.oh-my-zsh/custom/babun.zsh-theme
PROMPT='%{$fg[blue]%}{ %c } \
%{$fg[green]%}$(  git rev-parse --abbrev-ref HEAD 2> /dev/null || echo ""  )%{$reset_color%} \
%{$fg[red]%}%(!.#.>>)%{$reset_color%} '

5.3 修改PS1值

  • 先确认机器使用的shell
{ ~ }  » env | grep "SHELL"
SHELL=/bin/bash
  • 由于使用的是bash shell，只需改动bashrc配置
vim ~/.bashrc
export PS1='[\u@XXX_XXX_XXX.XXX.XXX.XXX_XXXX_A \W]\$ '
source ~/.bashrc

六、其他问题

6.1 修复SecureCRT 7.1无法登录

  • 解决方案
# 在win机上执行以下语句
ssh-keygen -t rsa -f /etc/ssh_host_rsa_key 
ssh-keygen -t dsa -f /etc/ssh_host_dsa_key 
chmod 600 /etc/ssh_host_rsa_key 
chmod 600 /etc/ssh_host_dsa_key

6.2 nfs挂载问题

  • ​挂载Linux共享出来的nfs目录，可以新建文件，但无法追加写文件。这个问题暂无解，后面改用了samba来处理

6.3 samba挂载只能本地终端可见，ssh连接过去无法识别到共享

  • 通过cygwin去挂载远程共享时，需要检查一下sshd服务启动的用户是否为当前配置连接的用户名（默认是cyg_server）。如果不是，需要修改为当前连接的用户名，如mc_win。
  • 修改方法：右键我的电脑 -> 管理 -> 服务 -> 双击CYGWIN sshd -> 登录选项卡

6.4 旧机器使用新用户启动ssh服务

  • windows 下创建新的m16用户
  • 使用windows用户本地认证
mkgroup -l > /etc/group
mkpasswd -l > /etc/passwd
  • 修改ssh相关文件归属
chown m16:Administrator /etc/ssh*
chown m16:None /var/empty
  • 拷贝ssh_key认证相关文件到m16家目录
cp -a /home/m16_fb/.ssh /home/m16/
chown m16:None -R /home/m16/.ssh
chmod 755 /home/m16

6.5 Erlang 安装路径处理

  • 由于项目脚本默认erlang路径位置：/cygdrive/c/Program\ Files/
  • 若Erlang安装在 /cygdrive/c/erl6.4 需要做『软链接』处理
ln -s /cygdrive/c/erl6.4 /cygdrive/c/Program\ Files/

6.6 PHP版本问题

  • 利用pact安装的php版本是5.6.26，而我们需求的是5.6.25，需要手动安装
  • 安装步骤如下：
cd ~
mkdir downloads
cd downloads
wget http://mirrors.163.com/cygwin/x86/release/php/php-5.6.25-1.tar.xz
wget http://mirrors.163.com/cygwin/x86/release/php/php-curl/php-curl-5.6.25-1.tar.xz
cd /
tar xf ~/downloads/php-5.6.25-1.tar.xz
tar xf ~/downloads/php-curl-5.6.25-1.tar.xz
# 确认php模块
php -m |egrep 'json|curl'
